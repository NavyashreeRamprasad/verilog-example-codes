This is because a vector is treated as a single binary value or bit-string. In contrast, an array (e.g., reg [7:0] mem[0:15]) is a collection of vectors and does not support operations on the entire array as a whole; operations like mem1 = mem2, mem1 + mem2, or mem1 == mem2 are not allowed. However, when accessing a single element of the array (e.g., mem[i]), that element behaves like a vector, and all operations become possible on it. Additionally, in Verilog, the same operator symbol can have different meanings depending on context—for example, & acts as a bitwise AND when used between two vectors (a & b) but as a reduction AND when applied to a single vector (&a), illustrating how operator behavior depends on usage.

9)1. Vectors (Packed)
	Declared as: reg [7:0] my_vector;
	Single Entity: Verilog treats this as a single multi-bit number.
	Operators: You can use all operators (+, -, &, |, <<, etc.) directly on the variable.
	Assignment: You can assign a value to the whole vector at once: my_vector = 8'hA5;.

	2. Arrays (Unpacked / Memories)
	Declared as: reg my_array [7:0]; (Note the range is after the name).
	Collection of Scalars: Verilog treats this as a group of separate variables (often called a "memory").
	Operators: Almost all operators are forbidden on the array as a whole. You cannot do my_array + 1 or if (my_array1 == my_array2).
	Assignment: You cannot assign an entire array at once. You must assign each element individually using an index:
	my_array[0] = 1'b1;
	my_array[1] = 1'b0;

	No Comparison: Even if two arrays are identical, you cannot compare them without using a for loop to check every single index.

15) a) y = -1 ? 10 : 20;
	Result: y = 10
	Reason: In Verilog, any non-zero integer is treated as logical TRUE. Since -1 is non-zero, the first expression (10) is selected.
	b) y = (-1 == 1) ? 10 : 20;
	Result: y = 20
	Reason: The expression -1 == 1 is FALSE. Therefore, the second expression (20) is selected.
	c) y = (-1 == -1) ? 10 : 20;
	Result: y = 10
	Reason: The expression -1 == -1 is TRUE, so the first expression (10) is selected.

i)Concise Code
Instead of writing multiple lines of procedural code, you can describe logic in a single line. This is particularly useful for simple assignments and signal routing.

ii)Flexible Placement (assign and always blocks)
One of the biggest advantages of the conditional operator is where it can be used:
In assign statements: It is used for continuous assignment to model combinational logic outside of procedural blocks.
In always blocks: It can also be used inside procedural blocks (like always @(*)) to assign values to reg types.
The if-else Constraint: As your image correctly notes, if-else statements are procedural. They can only be used inside initial or always blocks. They cannot exist in a continuous assignment.

iii) Why assign y = if(a) b else c; fails:
In Verilog, an assign statement expects an expression on the right-hand side. if-else is a statement—a structural command that directs the flow of execution—not a value-returning expression.
Conversely, a ? b : c is an expression that resolves to a specific value, making it compatible with the assign keyword
